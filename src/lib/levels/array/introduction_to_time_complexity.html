What is Time Complexity?

Time complexity is the measurement of the amount of time an algorithm takes to complete as a function of the size of its input.

To understand the entire premise of time complexity, we will break it down from the simple cases.

Runtime

Runtime is the amount of time an algorithm takes to complete. It is usually measured in seconds or milliseconds, but for the sake of simplicity, we will measure it in terms of the number of operations performed.

For example, for a simple one-line operation, we can say that the runtime is 1.

a = 5;

The following code snippet would have a runtime of 3:

a = 5;

b = 10;

c = a + b;

A for-loop with 10 iterations with one operation will have a runtime of 10:

for i in range(10):

    a += 1;

What if the number of iterations is not fixed? For example, if we have a for-loop that iterates over an array:

for i in range(len(arr)):

    a += 1;

In this case, the runtime is not fixed. It depends on the size of the array. If the array has 10 elements, the runtime is 10. If it has 100 elements, the runtime is 100. So in general, if an array has n elements, we can say that the runtime is n.

If the for loop has 3 operations, then the runtime is 3n.

for i in range(len(arr)):

    a += 1;

    b += 1;

    c += 1;

Relating Back to Time Complexity

When it comes to evaluating an algorithm's time complexity, we are not concerned with the actual runtime. We are concerned with how the runtime grows as the size of the input grows.

For the one-line example, we can say that the time complexity is O(1) because the runtime does not grow with the size of the input. It is constant.

Another example of an O(1) operation is getting the first element of an element: a = arr[0]. Regardless of the size of the array, this operation will always take the same amount of time.


For the for-loop example, we say that the time complexity is O(n) because the runtime grows linearly with the size of the input. If the array has n elements, the runtime is n.


For a double for-loop example:

for i in range(len(arr)):

    for j in range(len(arr)):

        a += 1;

we say that the time complexity is O(n2) because the runtime grows quadratically with the size of the input. If the array has n elements, the runtime is n2.

Converting Runtime to Time Complexity

When the runtime of a particular algorithm ends up being dependent on multiple instances of n, such as the following:

for i in range(len(arr)):

    for j in range(len(arr)):

        for k in range(len(arr)):

            a += 1;

            b += 1;

for i in range(len(arr)):

    for j in range(len(arr)):

        a += 1;

        b += 1;

        c += 1;

for i in range(len(arr)):

    a += 1;

a += 1;

The runtime is 2n3 + 3n2 + n + 1. When it comes to time complexity, we can ignore the coefficients and the lower order terms, so we can say that the time complexity is O(n3).

The reason we ignore the coefficients and lower order terms is because as n becomes very large, the lower order terms become insignificant. For example, if n = 1,000,000, then 2n3 + 3n2 + n + 1 is approximately equal to 2n3. But if n = 1,000,000,000, then 2n3 + 3n2 + n + 1 is approximately equal to 2n3. So we can say that the time complexity is O(n3).

A common connection that is made is with algebraic functions. For example, if we have a function f(x) = 2x3 + 3x2 + x + 1, we classify the function as a cubic function because the highest order term is x3. We classify the time complexity of algorithms the same way.

Other Common Classifications of Time Complexity

Logarithmic Time (O(log n))

Logarithmic time complexity is when the runtime grows logarithmically with the size of the input. This is common in algorithms that divide the input in half at each step, such as binary search.

Linearithmic Time (O(n log n))

Linearithmic time complexity is when the runtime grows linearly with the size of the input multiplied by the logarithm of the size of the input. This is common in algorithms that use divide and conquer, such as merge sort.

Polynomial Time (O(nk))

Polynomial time complexity is when the runtime grows polynomially with the size of the input. This is common in algorithms that have nested loops, such as bubble sort.

Exponential Time (O(kn))

Exponential time complexity is when the runtime grows exponentially with the size of the input. This is common in algorithms that have multiple recursive calls, such as the Fibonacci sequence.

Factorial Time (O(n!))

Factorial time complexity is when the runtime grows factorially with the size of the input. This is common in algorithms that generate all possible permutations of a set, such as the traveling salesman problem.
