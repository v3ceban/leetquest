Transfering Information Through Recursion

Recursion is a powerful tool in programming that allows us to solve problems by breaking them down into smaller, more manageable subproblems. In this section, we will explore how recursion can be used to transfer information between the subproblems and the main problem above it.

A Familiar Example

Consider the factorial function, which is defined as:

def factorial(n):

    if n == 1:

        return 1

    else:

        return n * factorial(n - 1)

In this example, the recursive function factorial calculates the factorial of a number n by multiplying it with the factorial of n - 1. The base case is when n is equal to 0, at which point the function returns 1.

As we can see, the information (the factorial value) is transferred back up the recursion stack as each recursive call returns its result.

For example, if we called factorial(5), the function would execute as follows:

factorial(5) = 5 * factorial(4)

factorial(5) = 5 * 4 * factorial(3)

factorial(5) = 5 * 4 * 3 * factorial(2)

factorial(5) = 5 * 4 * 3 * 2 * factorial(1)

factorial(5) = 5 * 4 * 3 * 2 * 1

factorial(5) = 120

As we can see, the information is transferred back up the recursion stack as each recursive call returns its result.

Transferring Information in Tree Traversals

In tree traversals, we can also transfer information between the subproblems and the main problem above it. For example, consider a binary tree where we want to calculate the sum of all the nodes in the tree.

We can use recursion to traverse the tree and calculate the sum of all the nodes:

def sum_tree(node):

    if node is None:

        return 0

    else:

        return node.val + sum_tree(node.left) + sum_tree(node.right)

In this example, the recursive function sum_tree calculates the sum of all the nodes in the tree by adding the value of the current node to the sum of the left and right subtrees. The base case is when the node is None, at which point the function returns 0.

Most of the problems following this will utilize recursion in this way, where we will be transferring information from the subproblems to the main problem above it.
