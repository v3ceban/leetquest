There Are Two Now?

This subconcept of two pointers will be significantly different from the ones you have seen so far. Instead of two pointers iterating over the same data structure, we will be using two pointers to iterate over two different data structures. This is a common technique used in problems that involve comparing or merging two structures.

Example (Intersection of Two Sorted Arrays)

Given two sorted integer arrays nums1 and nums2, return an array of their intersection. Each element in the result must appear as many times as it shows in both arrays, and you may return the result in any order.


This problem may look familiar. In fact, it is the same problem as the one in the Hash Map world, except the arrays are sorted. Since the arrays are sorted, we don't need to use a hash map to solve this problem.

We can use two pointers i and j to iterate through both arrays and compare the elements. If the elements are equal, we add them to the result array. If one element is smaller than the other, we move the pointer of the smaller element forward.
